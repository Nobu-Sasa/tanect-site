# ペットのあゆみ LP セクション — 作業引き継ぎ

Tanect2 の「ペットの様子」機能を Tanect-site の LP に追加する作業の引き継ぎ資料。
経営セッション側で **コード追加とビルド確認までは完了済み**。本セッションの主な仕事は
**実機スクリーンショットの差し替え・プレビュー確認・コミット**。

---

## 1. 対象機能の事実（コピー文言を書き換える際の必須ガードレール）

LP の文言は Tanect2 の実装に忠実であること。以下は実装済みの事実：

- 機能名は **「ペットの様子」**（ホーム画面のボタン）
- 入力フロー：ペット選択 → **写真撮影/選択（必須・動画ではない）** → 状況チップ（散歩中/ごはん/お昼寝/おやすみ/お留守番/遊んでる）＋自由文 → 位置情報（任意）
- AI は **Azure OpenAI GPT-4o Vision**。写真＋状況＋「最近の家族のできごと」から生成
- 出力は **「様子を解説」（第三者目線「〜しているようです」）のみ**
- **「気持ちを翻訳」（一人称セリフ）は誇張懸念で一旦非表示** ← LP でも触れない
- 安全設計：**健康断定を絶対禁止**。結果に必ず「これは AI による想像のおしゃべりです」を併記
- 保存後、みんなのフィード・今週の写真・週次レポートに自動統合

### 使ってよい訴求
写真を撮るだけ / AI がやさしく解説 / 家族みんなで見守る / 誇張しない安心設計

### 使ってはいけない文言（実装・安全思想と不一致）
- ❌「ペットがしゃべる」「気持ちを翻訳」
- ❌「健康チェック」「健康管理」
- ❌「散歩ルート地図」「短い動画」

## 3. 残タスク

### タスク A：実機スクリーンショットの差し替え（メイン）

現状 `PetStories.astro` の `.pet-shot` は「画面イメージ準備中」の CSS プレースホルダー枠。
nobu が Tanect2 の PetVoiceDialog の実画面（写真＋解説が出ている状態）を撮影して提供する。

差し替え手順：
1. 提供された画像を `public/screens/pet-voice.png` として配置（9:16 縦向き、電話フレーム込みでも枠なしでも可）
2. `PetStories.astro` の `<figure class="pet-shot">` 内のプレースホルダー枠を実画像に置換：
   ```astro
   <figure class="pet-shot">
     <img
       class="pet-shot-img"
       src={`${import.meta.env.BASE_URL.replace(/\/$/, '')}/screens/pet-voice.png`}
       alt="ペットの様子を解説している画面"
       loading="lazy"
       decoding="async"
       width="260"
     />
   </figure>
   ```
3. `.pet-shot-frame` / `.pet-shot-emoji` / `.pet-shot-note` の CSS を削除し、代わりに：
   ```css
   .pet-shot-img {
     width: 100%;
     height: auto;
     border-radius: var(--radius-lg);
     box-shadow: var(--shadow-md);
     display: block;
   }
   ```
   ※ 他の既存スクショ（`public/screens/*.png`）の扱いに合わせること。
4. **画像がまだ無い場合はプレースホルダーのまま据え置き**（壊れた img より良い）
